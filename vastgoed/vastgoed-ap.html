Dit document beschrijft een applicatieprofiel voor Vastgoed, in dit geval Vastgoed Basis AP. Een applicatieprofiel is een specificatie die bestaande termen hergebruikt van verschillende basis-standaarden en daar meer specificiteit aan toevoegt. Daarbij wordt het gebruik van de termen toegelicht voor de interbestuurlijke uitwisseling van overheidsinformatie. Het AP beantwoordt de vraag over hoe het corresponderende domeinmodel in de praktijk kan toegepast worden. Daarbij worden de beperkingen (kardinaliteit, codelijsten) toegelicht en de overeenkomstige (RDF) termen opgelijst.

1. Licentie

Deze specificatie van Informatie Vlaanderen is gepubliceerd onder de "Modellicentie Gratis Hergebruik - v1.0".

2. Conformiteit

Een applicatieprofiel is een specificatie voor gegevensuitwisseling dat bijkomende beperkingen introduceert voor het toepassen van vocabularia. Dergelijke bijkomende beperkingen kunnen de volgende elementen bevatten:

  • Verfijning van de terminologie (definitie van klassen en eigenschappen) consistent met de semantiek uit de betreffende specificaties met een welbepaald gebruik als doel;
  • Het bepalen van kardinaliteiten voor eigenschappen.
  • Externe terminologie (klassen en eigenschappen) gebruikt voor nieuwe/extra termen die niet in de bestaande vocabularia voorkomen.
  • Het gebruik van codelijsten.

Om conform te zijn met dit applicatieprofiel, MOET een implementatie:

  • Voor elke klasse steeds de eigenschappen bevatten die als minimum kardinaliteit 1 hebben.
  • Niet meer dan 1 instantie bevatten van eigenschappen die 1 als maximum kardinaliteit hebben.
  • De terminologie (klassen en eigenschappen) gebruiken op een manier die consistent is met haar semantiek (definitie, gebruik, domein en bereik).
  • Geen terminologie van andere gecontroleerde vocabularia gebruiken dan diegene gedefinieerd in dit applicatieprofiel.
  • Mag uitgebreid worden met klassen en eigenschappen van andere datamodellen (vocabularia) die niet overlappen met terminologie uit dit applicatieprofiel.

Gebouw

Een gesloten en/of overdekt, bovengronds of ondergronds bouwwerk, dat dient of bestemd is, ofwel om mensen, dieren en voorwerpen onder te brengen, ofwel om economische goederen te vervaardigen of diensten te verstrekken. Een gebouw verwijst naar gelijk we

Voor deze entiteit zijn de volgende attributen gedefinieerd: bouwjaar, geometrie, isOnderliggendPerceel.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

Gebouweenheid

De kleinste eenheid binnen een gebouw die geschikt is voor woon-, bedrijfsmatige, of recreatieve doeleinden en die ontsloten wordt via een eigen afsluitbare toegang vanaf de openbare weg, een erf of een gedeelde circulatieruimte. Een gebouweenheid is in functioneel opzicht zelfstandig. Daarnaast kan een gebouweenheid ook een gemeenschappelijk deel zijn.

Voor deze entiteit zijn de volgende attributen gedefinieerd: adres, functie, gebouweenheidStatus, geometrie, isOnderliggendPatrimoniaalPerceel, isFunctioneleEenheidBinnen.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
adres Adresvoorstelling 1 .. 1 Verwijst naar de adresnotatie voor dit object.
functie FunctieCodelijst 1 .. 1 Functie waarvoor de gebouweenheid in de realiteit (zoals vastgesteld op het terrein) gebruikt wordt.
gebouweenheidStatus GebouweenheidStatusCodelijst 1 .. 1 Fase van de levensloop waarin de gebouweenheid zich bevindt
geometrie Punt 1 .. 1 Associeert het object met de bijhorende geometrie.
isOnderliggendPatrimoniaalPerceel PatrimoniaalPerceel 0 .. * Verwijst naar een Patrimoniaal Perceel waar de Gebouweenheid deel van uitmaakt.
isFunctioneleEenheidBinnen Gebouw 1 .. 1 Verwijst naar het Gebouw waar de Gebouweenheid een functionele eenheid van uitmaakt.

  "naam": "NietGekend"

},

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst

onbebouwd OnbebouwdeGrond 0 .. * Verwijst naar een onbebouwde grond vervat in het Patrimoniaal Perceel.
perceel Perceel 1 .. 1 Verwijst naar het kadastraal planperceel waar het Patrimoniaal Perceel een eigendomseenheid van vormt.
gebouweenheid Gebouweenheid 0 .. * Verwijst naar de Gebouweenheid die deel uitmaakt van het Patrimoniaal Perceel.

Adresvoorstelling

Meer leesbare voorstelling met enkel de basisgegevens van het adres, bedoeld voor het gebruik van een adres als attribuut van een ander object.

Voor dit datatype zijn de volgende attributen gedefinieerd: administratieveEenheidNiveau1, administratieveEenheidNiveau2, adresgebied, busnummer, gemeentenaam, huisnummer, land, locatieaanduiding, locatienaam, postbus, postcode, postnaam, straatnaam, volledigAdres.

Eigenschap Verwacht Type Kardinaliteit Beschrijving Gebruik Codelijst
administratieveEenheidNiveau1 String 0 .. 1 Hoogste Administratieve eenheid vh adres, doorgaans een land.
administratieveEenheidNiveau2 String 0 .. 1 De regio vh adres, doorgaans een provincie of deelstaat of gelijkaardig gebied dat typisch meerdere plaatsen omvat.
adresgebied String 0 .. 1 Naam ve geografisch gebied of plaats die een aantal adresseerbare objecten groepeert om deze te adresseren zonder dat het gebied of de plaats een administratieve eenheid is Bvb de naam vh gehucht waarin het adres ligt.
busnummer String 0 .. 1 Officieel toegekende alfanumerieke code die wordt toegevoegd aan het huisnummer om meerdere gebouweenheden, standplaatsen, ligplaatsen of percelen te onderscheiden die eenzelfde huisnummer hebben. Specialisatie van Adresvoorstelling:locatieaanduiding tbv Belgische adressen.
gemeentenaam String 0 .. 1 Gemeentenaam vh adres.
huisnummer String 0 .. 1 Alfanumerieke code officieel toegekend aan gebouweenheden, ligplaatsen, standplaatsen of percelen. Specialisatie van Adresvoorstelling:locatieaanduiding tbv Belgische adressen.
land String 0 .. 1 Land waarin het adres gelegen is.
locatieaanduiding String 0 .. 1 Alfanumerieke code die de adreslocator uniek identificeert binnen de straat, administratieve eenheid etc.
locatienaam String 0 .. 1 Naam of omschrijving vh het geografisch object dat de adreslocator aanduidt.
postbus String 0 .. 1 Postbusnummer in een postkantoor.
postcode String 0 .. 1 Code waarmee het geografisch gebied dat adressen voor postale doeleinden groepeert wordt aangeduid.
postnaam String 0 .. 1 Naam waarmee het geografisch gebied dat adressen voor postale doeleinden groepeert wordt aangeduid.
straatnaam String 0 .. 1 Straatnaam vh adres.
volledigAdres String 0 .. 1 Het complete adres in één string, al dan niet geformatteerd. Vermijdt fouten tgv het opsplitsen ve adres in zijn onderdelen. Geeft de voorgeschreven volgorde vd verschillende onderdelen weer.

          "naam": "OUD-HEVERLEE"

      }
